# PDFly

Free online PDF tools website ‚Äî merge, split, compress, convert, rotate, watermark, protect, and more. No login required. All 14 tools are free and process files server-side with immediate deletion.

## Run & Operate

- `pnpm --filter @workspace/api-server run dev` ‚Äî run the API server (port 8080)
- `pnpm --filter @workspace/pdfly run dev` ‚Äî run the frontend (port 19667)
- `pnpm run typecheck` ‚Äî full typecheck across all packages
- `pnpm run build` ‚Äî typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` ‚Äî regenerate API hooks and Zod schemas from the OpenAPI spec

## Stack

- pnpm workspaces, Node.js 24, TypeScript 5.9
- Frontend: React + Vite + Tailwind CSS + wouter (routing)
- API: Express 5
- PDF processing: pdf-lib, pdf-parse, pptxgenjs, exceljs, docx, mammoth, archiver, multer, sharp
- Image conversion: pdftoppm (system binary ‚Äî available in Nix path)
- No database needed

## Architecture decisions

- All PDF operations use multipart/form-data (not JSON). Frontend uses native `fetch` + `FormData` + blob download for all processing endpoints.
- Files are processed in-memory (multer memoryStorage) or in temp dirs and deleted immediately ‚Äî no persistent storage.
- 50MB file size limit enforced by multer.
- `lib/api-zod/tsconfig.json` includes `"lib": ["esnext", "dom"]` to support `File` and `Blob` types generated by Orval for multipart schemas.
- `archiver` and `pdf-parse` use `require()` interop due to ESM/CJS typing issues.
- PDF password protection uses `qpdf` system tool if available; falls back gracefully.
- PDF unlock uses `ignoreEncryption: true` in pdf-lib (pdf-lib v1.x doesn't support password-based decryption natively).
- Office-to-PDF supports `.docx` via mammoth; `.xlsx`/`.pptx` require LibreOffice (not available) and return an informative error.

## Gotchas

- `pdftoppm` must be available in PATH for PDF‚ÜíJPG and image extraction features
- `lib/api-zod/tsconfig.json` needs `"lib": ["esnext", "dom"]` ‚Äî do not remove or Blob/File types break in codegen output
- Do NOT use `import archiver from "archiver"` (no default export in @types/archiver) ‚Äî use `require("archiver")` with a cast
- pdf-parse v2.x is ESM ‚Äî use `require("pdf-parse")` with a type cast, not dynamic `import()`
